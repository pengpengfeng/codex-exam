# MVP摘要
交付企业内网私有化在线考试系统（并发≤500），支持员工/外包/实习使用账号密码登录考试、自动+人工阅卷、错题解析与电子证书；不包含AI监考、多租户子公司和超500并发能力。

## 1. 用户角色
| 角色 | 核心目标 | 使用频率 | 专业程度 |
|---|---|---|---|
| 考生（员工/外包/实习） | 在规定时间完成考试并查看结果/证书 | 每月1-4次 | 低-中 |
| 出题人/培训专员 | 维护题库、配置考试模板、发布考试 | 每周1-3次 | 中 |
| 阅卷老师 | 复核主观题并确认最终成绩 | 每周1-2次 | 中 |
| 部门管理员 | 组织考试、跟踪通过率、导出审计数据 | 每周1-2次 | 中 |
| 系统管理员 | 账号权限、参数配置、审计留存策略 | 每周1次 | 中-高 |

## 2. 用户故事
1. 作为考生，我希望使用账号密码登录并参加考试，以便完成培训、合规和晋升认证。
2. 作为出题人，我希望可配置固定卷+随机卷混合组卷，以便平衡标准化与防记忆化。
3. 作为阅卷老师，我希望系统先自动判分再人工复核主观题，以便提高效率并保证评分准确。
4. 作为部门管理员，我希望按考试模板配置及格线并发布结果+证书，以便满足不同考试政策。
5. 作为审计负责人，我希望保留5年考试与答题记录并可导出审计报表，以便满足内控与合规检查。

## 3. 验收标准
### 用户故事1：登录与考试
- [ ] 仅支持账号密码登录，不提供第三方统一认证入口。
- [ ] 登录成功后考生仅可见本人可参加考试列表，禁止越权访问他人考试。
- [ ] 考生可在考试时间窗内进入考试；超时自动交卷并记录交卷时间。

### 用户故事2：组卷能力
- [ ] 支持配置“固定题目+随机抽题”混合规则（按题型、难度、题量）。
- [ ] 同一考试批次不同考生题目顺序随机，且题目选项顺序可随机。
- [ ] 发布后的考试模板需版本化，已开考批次不可被覆盖修改。

### 用户故事3：阅卷评分
- [ ] 客观题自动判分，主观题进入人工复核队列。
- [ ] 人工复核支持二次评分与评分备注，并保留操作人和时间戳。
- [ ] 最终成绩=客观题得分+主观题复核得分，成绩发布前可回溯评分明细。

### 用户故事4：通过规则与发证
- [ ] 每个考试模板可独立配置及格线（0-100，精确到1分）。
- [ ] 成绩发布后考生可查看总分、是否通过、错题解析。
- [ ] 通过考生可下载电子证书（含姓名、考试名、成绩、发证时间、唯一证书号）。

### 用户故事5：审计与留存
- [ ] 试卷、作答、评分、发布、证书发放日志默认留存5年。
- [ ] 支持按考试/部门/时间范围导出审计报表（CSV/XLSX）。
- [ ] 所有导出行为记录操作人、时间、筛选条件并可追溯。

## 4. 功能需求
1. 身份认证与权限
   - 仅支持账号密码登录通道。
   - 角色：考生、出题人、阅卷老师、部门管理员、系统管理员。
   - 基于角色的页面与接口权限控制（RBAC）。
2. 题库与考试模板
   - 题型：单选、多选、判断、填空、简答。
   - 支持题目难度、标签、知识点分类管理。
   - 模板支持固定+随机混合组卷，支持版本化。
3. 考试执行
   - 考试时间窗、答题时长、自动交卷。
   - 中等防作弊：题目乱序、切屏记录、全程限时、禁复制。
   - 异常中断后可在策略允许范围内续考。
4. 阅卷与成绩
   - 客观题自动评分，主观题人工复核。
   - 评分流程支持复核备注与历史留痕。
   - 结果发布后生成错题解析与电子证书。
5. 报表与审计
   - 管理端查看考试完成率、通过率、平均分。
   - 审计导出：考试记录、答题明细、评分日志、证书日志。
   - 数据按5年留存策略归档与检索。
6. 系统管理
   - 登录策略、密码策略、考试策略参数化配置。
   - 组织架构/人员同步（至少支持批量导入）。

## 5. 非功能需求（量化）
1. 性能
   - 峰值并发考生≤500时，提交答卷接口P95响应<1.5秒。
   - 常规页面查询P95响应<1.0秒；报表导出任务排队等待<30秒。
2. 可扩展性
   - 支持横向扩展到2个应用实例且功能无变更。
   - 题库容量支持≥20万题，单场考试最大题量200。
3. 权限与安全
   - 密码最少12位并含大小写+数字+特殊字符。
   - 管理员操作与导出操作100%写入审计日志。
   - 敏感字段（身份证号/手机号）存储加密，传输使用TLS。
4. 易用性
   - 考生端关键流程（登录→开考→交卷）3步内完成。
   - 管理端新建考试模板平均操作时长<10分钟（基于30题）。
5. 可维护性
   - 关键业务模块具备可观测日志与错误码。
   - 核心流程（登录、开考、交卷、评分、发证）具备自动化回归用例。

## 6. 优先级与MVP范围
### P0（MVP必须）
- 账号密码登录（单一登录方式）
- 5类题型（单选/多选/判断/填空/简答）
- 固定+随机混合组卷
- 中等防作弊（乱序、切屏记录、限时、禁复制）
- 自动评分+主观题人工复核
- 模板级及格线
- 错题解析+电子证书
- 5年留存与审计导出
- 内网私有化部署、并发≤500

### P1（下一阶段）
- 更细粒度组织同步（实时对接HR系统）
- 报表中心（趋势分析、部门对比看板）
- 证书验真公开查询页（仅内网）

### P2（范围外）
- AI监考（摄像头行为识别）
- 多租户子公司隔离架构
- 并发>500的大规模弹性扩容
- 外部互联网公开考试

## 7. 待解决问题与风险
1. 账号体系来源（自建/LDAP/AD同步）未确认（风险：登录改造延期）。
2. 主观题评分SLA未定义（风险：成绩发布时间不可控）。
3. 5年留存的存储成本与归档策略未确认（风险：运维成本超预算）。
4. 电子证书法律/内控效力要求未确认（风险：证书不可用于正式审计）。
5. 内网部署环境基线（OS、数据库、中间件版本）未确认（风险：上线窗口延迟）。
